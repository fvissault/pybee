Interpréteur : généralités

Système de fichiers | Index | Interpréteur : notions de séquences

Beetle est un langage interprété écrit en Python. Un interpréteur est donc écrit pour exécuter vos mots. L'interpréteur se base principalement sur 2 piles LIFO : pile de travail (work stack) et pile de retour (return stack). Il vous permet d'exécuter du code Beetle en empilant un certain nombre de séquences d'instructions. Ceci est expliqué dans le chapitre suivant.

L'interpréteur Beetle va interpréter l'ensemble des mots que vous taper dans la console. Si vous commettez une erreur de syntaxe, l'interpréteur va vous expliquer quoi et ou votre erreur a été faite.

Le package 'core' est automatiquement chargé dans l'interpréteur. Vous ne pouvez pas charger 2 fois le package 'core'. Les mots du package 'core' sont attribués à l'interpréteur par défaut. Celui-ci va vous permettre :

  • De créer des définitions
  • D'interpréter des conditionnelles
  • D'interpréter des boucles
  • D'interpréter des instructions Beetle en général
  • D'obtenir de l'aide
  • D'inclure des données simples comme des entiers, des nombres décimaux, etc... dans la pile de données (work stack)

L'interpréteur ne fait que reconnaitre :

  • Les entiers (1, 12, -5, …)
  • Les nombres décimaux (1.2, 1e-10, 14E15, …)
  • Les chaines de caractères ( "chaine de caractères", …) Seul le caractère " délimite les chaines de caractères
  • Et les mots du dictionnaire

Tous les mots doivent être séparés par le caractère ESPACE

2 types de comportement sont autorisés par l'interpréteur. Dans les séquences :

  • soit la notation polonaise inversée est utilisée
  • soit la séquence d'instructions est utilisée

Dans le cas ou la notation polonaise inversée est utilisée, les instructions vont lire la pile de données (work stack)
Par exemple :

2 dup dump => affiche 2 2

Dans le cas ou on utilise la séquence d'instructions, le sujet est placé après le mot comme dans l'exemple ci-dessous :

load my_file => exécute le contenu de my_file.btl

Cette écriture est plus simple car si l'on devait utiliser la pile de travail (work stack) pour interpréter cette instruction, nous devrions écrire quelque chose comme cela :

"my_file" load ou my_file load

Ce qui est moins intuitif et donc moins clair. Dans la deuxième écriture, l'interpréteur renvoie un erreur qui indique que 'my_file' n'est pas un mot connu.
