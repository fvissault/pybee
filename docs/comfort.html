X.3. Interpreteur - Mots de confort

Précédent : X.2. Interpreteur - Notions de séquence | Index | Suivant : XI.1. Structures du langage - Manipulation de la pile

Beetle vous offre la possibilité d'utiliser des mots de comfort. Généralement, ces mots (que nous allons passer en revue plus bas) sont des mots qui n'utilisent pas la pile de travail mais utilisent la suite de la séquence.

Load

Ce mot vous permet de charger le contenu d'un fichier btl et de l'interpréter. Le fichier btl doit être précisé sans l'extension btl.

Sa syntaxe est la suivante :

Beetle> load my_file

see

Ce mot vous permet de visualiser une définition si celle-ci n'est pas native. Si la définition est native, un warning vous l'indiquera

Sa syntaxe et son résultat sont les suivants :

Beetle> see dup
: dup
0 pick
;
ok

Beetle> see list
see list warning : nothing to see, it is a native definition
ok

list

Ce mot vous permet de visualiser le contenu d'un fichier btl sans l'interpréter. Le fichier btl doit être précisé sans l'extension btl.

Sa syntaxe et son résultat sont les suivants :

Beetle> list import
<?btl
import math
import file
import mysqldb
import date
packages
words
?>
ok

constants

Ce mot vous donne la liste des constantes disponibles.

Sa syntaxe et son résultat sont les suivants :

Beetle> constants
true = 1 ; false = 0 ; base = 10 ;
ok

variables

Ce mot vous donne la liste des variables disponibles.

Sa syntaxe et son résultat sont les suivants :

Beetle> variables
path = userarea ;
ok

packages

Ce mot vous donne la liste des packages disponibles.

Sa syntaxe et son résultat sont les suivants :

Beetle> packages
core : version = v1.5.5
ok

dump

Ce mot vous affiche la liste des données qui se trouvent sur la pile de travail.

Sa syntaxe et son résultat sont les suivants :

Beetle> 1 2 3 4 5 + dump
9 3 2 1 ok

?

Ce mot vous affiche le contenu d'une variable en particulier. Vous ne pouvez pas afficher le contenu d'une variable qui n'existe pas. C'est pour cela que sa syntaxe est la suivante.

Sa syntaxe et son résultat sont les suivants :

Beetle> path ?
userarea ok

words

Ce mot vous affiche dans la console tous les mots disponibles de votre dictionnaire.

Sa syntaxe et son résultat sont les suivants :

Beetle> words

wp

work pointer (wp) : Ce mot met sur la pile de travail l'adresse du prochain élément de la pile de travail. Si 0 est renvoyé, cela signifie que la pile de travail est vide

Sa syntaxe et son résultat sont les suivants :

Beetle> wp .
0 ok

sp

sequence pointer (sp) : Ce mot met sur la pile de travail l'adresse de la prochaine séquence de l'interpréteur. 0 ne sera jamais renvoyé.

Sa syntaxe et son résultat sont les suivants :

Beetle> sp .
1 ok

vp

vocabulary pointer (vp) : Ce mot met sur la pile de travail l'adresse du prochain mot dans le dictionnaire.

Sa syntaxe et son résultat sont les suivants :

Beetle> vp .
168 ok

?pack

Ce mot vous permet de savoir si un package a été chargé. La valeur renvoyée est soit 0 (false), soit 1 (true).

Sa syntaxe et son résultat sont les suivants :

Beetle> ?pack core .
1 ok

Beetle> ?pack other_pack_name .
0 ok

cls

clear stack (cls) : vide la pile de données

Sa syntaxe et son résultat sont les suivants :

Beetle> cls
ok

clt

clear terminal (clt) : vide la console

Sa syntaxe et son résultat sont les suivants :

Beetle> clt
ok

( ... )

Ces 2 mots '(' et ')' vous permettent d'écrire un commentaire dans vos programmes. Bien sûr, cela ne fait pas grand-chose mais cela deviendra utile quand vous écrirez des programmes plus complexes.

Sa syntaxe et son résultat sont les suivants :

Beetle> ( ceci est un commentaire )
ok
