XII.3. Module 'file'

Précédent : XII.2. Packages:Module 'math' | Index | Suivant : XII.4. Packages:Module 'date'

Le package 'file' est uniquement dédié à la manipulation de fichiers : ouverture, lecture, écriture et fermeture. A prioi, vous pouvez manipuler n'importe quel fichier se trouvant su votre serveur. A chaque fois que vous ouvrirez un fichier, un descripteur de fichier sera créé. C'est avec ce descripteur que vous pourrez avoir d'autres actions sur vos fichiers comme lire, écrire ou fermer le fichier que vous aurez ouvert.

1. Ensemble des mots

L'ensemble des mots qui composent ce package est le suivant :
appendtofile closefile overwritetofile readchar readfile readingfile readline writein

2. Ouverture de fichiers

Ouverture en lecture (readingfile)

Pour ouvrir un fichier en lecture seule, le mot à utiliser est 'readingfile'

La syntaxe de ce mot est :

"nom_de_la_variable_qui_contient_le_descripteur" "nom_du fichier_a_ouvrir" readingfile

Dans nom_de_la_variable_qui_contient_le_descripteur, nous retrouvons le descripteur du fichier qui manipule le fichier dont le nom est nom_du fichier_a_ouvrir

Ouverture en écriture au début du fichier (overwritetofile)

Pour ouvrir un fichier en écriture et effacer son contenu éventuel, le mot à utiliser est 'overwritetofile'

La syntaxe de ce mot est :

"nom_de_la_variable_qui_contient_le_descripteur" "nom_du fichier_a_ouvrir" overwritetofile

Ouverture en écriture à la fin du fichier (appendtofile)

Pour ouvrir un fichier en écriture sans effacer son contenu éventuel, le mot à utiliser est 'appendtofile'

La syntaxe de ce mot est :

"nom_de_la_variable_qui_contient_le_descripteur" "nom_du fichier_a_ouvrir" appendtofile

3. Lecture/écriture dans un fichier

Lecture d'un caractère (readchar)

Pour lire un caractère dans un fichier, le mot à utiliser est 'readchar'
Le caractère lu est déposé sur la pile de travail

La syntaxe de ce mot est :

"variable_qui_contient_le_descripteur" readchar

Lecture d'une ligne (readline)

Pour lire une ligne dans un fichier délimitée par un retour chariot, le mot à utiliser est 'readline'
La ligne lue est déposée sur la pile de travail

La syntaxe de ce mot est :

"variable_qui_contient_le_descripteur" readline

Lecture d'un fichier (readfile)

Pour lire un fichier en entier, le mot à utiliser est 'readfile'
Le contenu du fichier lu est déposé sur la pile de travail

La syntaxe de ce mot est :

"variable_qui_contient_le_descripteur" readfile

Ecriture dans un fichier (writein)

Pour écrire dans un fichier ouvert en écriture, le mot à utiliser est 'writein'

La syntaxe de ce mot est :

"contenu" "variable_qui_contient_le_descripteur" writein

4. Fermeture d'un fichier

Fermeture d'un fichier (closefile)

Pour fermer un fichier, le mot à utiliser est 'closefile'

La syntaxe de ce mot est :

"variable_qui_contient_le_descripteur" closefile
