V. Dictionnaires

Précédent : IV. Piles de données | Index | Suivant : VI. Packages

Le principe de ce langage est de s'appuyer sur un ensemble de mots que vous allez enrichir avec vos propres mots (ou définitions). Contrairement à un langage classique tel que C ou PHP ou même Python, vous êtes le constructeur de ce langage. Il vous faudra donc penser légèrement différemment pour réaliser vos applications. Pour faire une analogie très lointaine, un mot correspond à une fonction ou une procédure dans un autre langage. Si je peux vous donner une conseil, vos mots doivent réaliser des actions simples comme vous pourriez le faire avec le langage Lisp au risque de ne plus rien comprendre à ce que vous faites.

Beetle utilise donc un dictionaire de mots principal que nous appellerons 'core'. Ce dictionnaire comporte une liste assez conséquente de mots de base.

Vous avez à disposition d'autres dictionnaires que vous pouvez ajouter à votre dictionnaire principal. Aujourd'hui, voici la liste :

  • dictionnaire 'math' qui regroupe tous les mots nécessaires pour faire des mathématiques
  • dictionnaire 'date' qui vous permet de manipuler des dates
  • dictionnaire 'file' qui vous permet de manipuler des fichiers
  • dictionnaire 'mysqldb' qui vous offre la possibilité de vous connecter et requêter une base de données MySql
  • dictionnaire 'web' qui vous offre la possibilité de développer vos pages web

Tous ces dictionnaires sont indépendants et vous les trouverez dans des packages distincts. Pour les utiliser, vous devrez utiliser l'instruction 'import'

Pour créer une librairie de nouveaux mots et donc un nouveau dictionnaire, vous devez créer un package. Nous verrons la création de nouveaux packages dans le chapitre suivant.

Le mot qui vous affiche la liste des mots disponibles dans l'ensemble des mots importés est : 'words'
