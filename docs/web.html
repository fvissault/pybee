2. Principe général : structure de données et mots principaux (addattr, addcontent)

Structure de données

La seule structure de données manipulée par le package 'web' est la suivante :

{ (c'est un dictionnaire)
"tag" : "my_tag", (corresponds au nom du tag : html, head, body, ...)
"content" : list_of_content, (corresponds au contenu du container représenté par le tag)
"attrs" : attr_dict, (dictionnaire des attributs du tag)
"container" : "y|n" (y = c'est un container, n = ce n'est pas un container)
}

Exemple de structure de données définie avec Beetle :

{ tag : "input" , content : [ ] , attrs : attrs @ , container : "n" }

Cette structure pour définir un input HTML. Ce n'est pas un container donc il n'a pas de contenu et il n'a que des attributs. Les attributs se retrouvent dans la variable attrs.

Le package se charge de créer les tags principaux de la page web : comme 'html', 'head', 'body', 'title'. Ces structures sont stockées dans des variables qui portent le même nom : la structure pour le tag 'html' se trouve dans la variable html, etc...

Mots principaux

Pour ajouter un attribut à un tag, ce sera le mot : addattr. La syntaxe du mot est la suivante :

contenant_variable "attribut_name" "attribut_value" addattr

Pour ajouter un contenu à un tag, ce sera le mot : addcontent. La syntaxe du mot est la suivante :

contenant_variable contenu_variable addcontent

Exemple concret (vous pouvez voir que ces 2 mots sont très importants dans le package 'web') :

<?btl
webreset
body form addcontent
title "Page test de passage de paramètres dans un formulaire" addcontent
form "action" "test_form_result.btl" addattr
html "lang" "en" addattr
head "UTF-8" charset addcontent
head "userarea/css/file.css" headlink addcontent
form "Ceci est mon label : " "mydiv" label addcontent
"mydiv" "fred" "text" fieldarea var fa drop
fa "name" "faname" addattr
form fa addcontent
form br addcontent
form "submit" "Valider" button addcontent
html @ generate cls
forget fa
?>

Contenu : chaine de caractères

La syntaxe de ce mot est :

Tag : <audio>

Attributs obligatoires : controls, autoplay, inloop, muted

Si controls = 'controls' alors l'attribut controls apparaitra dans les attributs du tag sinon il n'apparait pas
Si autoplay = 'autoplay' alors l'attribut controls apparaitra dans les attributs du tag sinon il n'apparait pas
Si inloop = 'loop' alors l'attribut loop apparaitra dans les attributs du tag sinon il n'apparait pas
Si muted = 'muted' alors l'attribut muted apparaitra dans les attributs du tag sinon il n'apparait pas

Contenu : pas de contenu

La syntaxe de ce mot est :

"controls_attribut" "autoplay_attribut" "inloop_attribut" "muted_attribut" audio --> {'tag':'audio', 'content':[], 'attrs':{'muted':'muted_attribut', 'autoplay':'autoplay_attribut', 'loop':'inloop_attribut', 'controls':'controls_attribut'}, 'container':'y'} est positionné sur la pile
